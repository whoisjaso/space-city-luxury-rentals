<objective>
Create the shared public layout shell — a sticky header with the Space City logo and navigation, a mobile hamburger menu, and a reusable PublicLayout that wraps all non-landing public pages.

Purpose: Every public page (fleet, vehicle detail, booking, status) needs consistent branded navigation. The landing page keeps its own cinematic Hero nav, but all other public pages share this layout. This establishes the navigation architecture for Phases 4-6.

Output: PublicHeader, MobileMenu, PublicFooter (reuse existing Footer), PublicLayout — all wired into App.tsx with placeholder routes.
</objective>

<tasks>
<task id="1" name="Create PublicHeader with logo, nav links, and mobile menu" type="code">
Create the following components:

1. `app/src/components/PublicHeader.tsx`:
   - Fixed/sticky header at top with z-50
   - Background: transparent initially, then dark (#050505) with backdrop-blur on scroll (use a scroll listener or IntersectionObserver)
   - Left: Space City logo (h-10 w-10) linking to "/" + "SPACE CITY" text
   - Right: Nav links — Home (/), Fleet (/fleet), Experiences (/#experiences), Contact (/#contact)
   - Mobile: Hide nav links, show hamburger icon button
   - Use the brand colors: gold accents on hover, white/70 default text
   - Use museo-label class for nav text
   - Import Link from react-router for internal navigation
   - For hash links (/#experiences, /#contact), use regular <a> tags so Lenis handles them on the landing page

2. `app/src/components/MobileMenu.tsx`:
   - Full-screen or slide-out overlay menu for mobile
   - Dark background (#050505) with full opacity
   - Shows logo at top, nav links stacked vertically, close button
   - Smooth open/close animation (CSS transition or simple state toggle)
   - Links close the menu on click
   - Gold accent color for active/hover states

3. `app/src/layouts/PublicLayout.tsx`:
   - Wraps children with PublicHeader at top and Footer at bottom
   - Main content area has pt to account for fixed header
   - Dark background (#050505) as default
   - Import Footer from sections/Footer (reuse existing)

4. `app/src/App.tsx` — Update routing:
   - "/" → LandingPage (no PublicLayout, keeps its own nav)
   - "/fleet" → PublicLayout wrapping a placeholder FleetPage
   - "/fleet/:slug" → PublicLayout wrapping a placeholder VehicleDetailPage
   - "/book" → PublicLayout wrapping a placeholder BookingPage
   - "/book/:code" → PublicLayout wrapping a placeholder BookingStatusPage
   - "/admin" → placeholder (no layout yet, Phase 7)
   - "*" → Navigate to "/"
   - Create minimal placeholder page components inline or in pages/ directory:
     ```tsx
     function FleetPage() { return <div className="min-h-screen pt-24 px-8"><h1 className="text-white text-4xl">Fleet — Coming Soon</h1></div>; }
     ```

Responsive requirements:
- Mobile (375px): Hamburger menu, logo only in header, stacked nav in mobile menu
- Tablet (768px): Can show some nav links or hamburger
- Desktop (1440px): Full nav links visible, no hamburger
